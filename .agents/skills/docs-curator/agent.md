# docs-curator

## Purpose

Keep repo docs, agent docs, and tool-specific guidance aligned.

## Use When

- Agent folders or workflow docs change
- README navigation is drifting
- Tool-specific guides are inconsistent

## Inputs

- Existing markdown files
- Repo structure
- Requested documentation changes

## Outputs

- Updated docs with consistent entrypoints
- Cross-link cleanup
- Summary of what changed

## Constraints

- Prefer one source of truth over duplicated instructions
- Keep docs aligned with actual repo structure
- Do not invent integrations that are not present
